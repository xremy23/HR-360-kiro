DISASTER RECOVERY PROCEDURES
================================================================================

PROCEDURE 1: RESTORE FROM BACKUP
  1. Go to Cloud SQL instance page
  2. Click "Backups" tab
  3. Select backup to restore
  4. Click "Restore"
  5. Confirm restoration
  6. Wait for restoration to complete (5-10 minutes)
  7. Verify data integrity
  8. Update backend connection if needed

PROCEDURE 2: POINT-IN-TIME RECOVERY (PITR)
  1. Go to Cloud SQL instance page
  2. Click "Backups" tab
  3. Click "Create backup from point in time"
  4. Select recovery timestamp (within 7 days)
  5. Confirm recovery
  6. Wait for recovery to complete
  7. Verify data integrity
  8. Update backend connection if needed

PROCEDURE 3: CLONE INSTANCE FROM BACKUP
  1. Go to Cloud SQL instance page
  2. Click "Backups" tab
  3. Select backup to clone from
  4. Click "Clone instance"
  5. Configure new instance
  6. Confirm cloning
  7. Wait for cloning to complete
  8. Test new instance
  9. Update backend connection if needed

PROCEDURE 4: EXPORT TO CLOUD STORAGE
  1. Go to Cloud SQL instance page
  2. Click "Export" button
  3. Select database to export
  4. Choose export format (SQL or CSV)
  5. Specify Cloud Storage bucket
  6. Confirm export
  7. Wait for export to complete
  8. Download exported file if needed
