ROLLBACK PROCEDURE (IF NEEDED)
==================================

If deployment causes issues, rollback to previous version:

Option 1: Delete and redeploy
   gsutil -m rm -r gs://hr-360-web-app/*
   gsutil -m cp -r "path/to/backup/dist/*" gs://hr-360-web-app/

Option 2: Keep multiple versions
   gsutil -m cp -r "path/to/current/dist/*" gs://hr-360-web-app/v2/
   Then update CNAME/redirect to point to v2/

TROUBLESHOOTING
==================================

Issue: "403 Access Denied" when accessing app
Fix: Verify bucket public access:
   gsutil iam ch allUsers:objectViewer gs://hr-360-web-app/

Issue: "Old version still showing"
Fix: Clear CloudCDN cache:
   gcloud compute url-maps invalidate-cdn-cache web-app \
     --path=/*

Issue: "White screen / blank page"
Fix: Check browser console for errors
   - Right-click → Inspect → Console tab
   - Check for JavaScript errors
   - Check for 404 errors on files
   - Verify backend URL in .env

Issue: "Cannot login / magic link not working"
Fix: Check backend connectivity
   curl https://backend-116253736511.us-central1.run.app/health
   
If backend is down:
   - Check Cloud Run service status
   - Check backend logs: gcloud run logs read

Issue: "Email not received"
Fix: Check Gmail credentials
   - Verify SMTP settings in backend/.env
   - Check Gmail app password is correct
   - Try resending from backend logs

AUTOMATION
==================================

To automate deployment, create a batch script:

File: deploy-web.bat
   @echo off
   echo Building web app...
   cd web
   call npm run build
   if errorlevel 1 (
     echo Build failed!
     exit /b 1
   )
   cd ..
   
   echo Deploying to Google Cloud...
   gsutil -m cp -r "web\dist\*" gs://hr-360-web-app/
   
   echo Deployment complete!
   echo Access at: https://web-116253736511.us-central1.run.app

Then run:
   deploy-web.bat
