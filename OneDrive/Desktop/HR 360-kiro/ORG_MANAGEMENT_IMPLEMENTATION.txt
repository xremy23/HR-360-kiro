BACKEND IMPLEMENTATION DETAILS
================================================================================

FILE: backend/src/routes/organization.ts
STATUS: COMPLETE - Deployed in backend-00030-bhk

ENDPOINTS IMPLEMENTED:

1. GET /org
   - Retrieve current organization details
   - Authentication: Required
   - Returns: Organization object with name, emailDomain, inviteCode, logo, createdAt

2. POST /org (Create Organization)
   - Create a new organization
   - Creator automatically becomes admin
   - Parameters: name (required), emailDomain (optional), logo (optional)
   - Returns: Organization object with unique inviteCode
   - Authentication: Required

3. PUT /org (Update Organization)
   - Update organization details
   - Admin-only action
   - Parameters: name, logo
   - Returns: Updated organization object
   - Authentication: Required, Admin role

4. POST /org/invite (Send Invitation)
   - Invite user to organization
   - Sends email with invitation link
   - Parameters: email (required), role (optional, default: 'employee')
   - Valid roles: admin, hr, manager, employee
   - Returns: Confirmation with inviteCode and organization name
   - Authentication: Required, Admin role

5. DELETE /org/users/:userId (Remove User)
   - Remove user from organization
   - Cannot remove yourself
   - Admin-only action
   - Returns: Success confirmation
   - Authentication: Required, Admin role

6. POST /org/switch (Switch Organization)
   - User switches to different organization (future feature)
   - Parameters: orgId (required)
   - Authentication: Required

7. GET /org/teams (List Teams)
   - Get all teams in organization
   - Groups users by teamId
   - Returns: Array of team objects with member counts
   - Authentication: Required

8. GET /org/users (List Organization Members)
   - Get all users in organization
   - Admin-only action
   - Query params: teamId (filter), role (filter), limit, offset
   - Returns: Paginated user list
   - Authentication: Required, Admin role

AUTHORIZATION:
- Admin Middleware: Checks req.user.role === 'admin' or 'hr'
- Prevents unauthorized access to management endpoints
- Prevents users from removing themselves
- Prevents cross-organization access

EMAIL SERVICE:
- Method: sendInvitationEmail(email, orgName, inviteLink, invitedBy)
- Template: organizationInvitation
- Status: Configured and tested with Gmail SMTP
