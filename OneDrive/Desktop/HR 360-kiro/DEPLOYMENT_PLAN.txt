PHASE 1 DETAILED: CLOUD SQL SETUP
================================================================================

Step 1: Create Cloud SQL Instance
  Command: gcloud sql instances create hr-360-db \
    --database-version=POSTGRES_15 \
    --tier=db-f1-micro \
    --region=us-central1 \
    --availability-type=REGIONAL

Step 2: Create Database
  Command: gcloud sql databases create emergency_app \
    --instance=hr-360-db

Step 3: Create Database User
  Command: gcloud sql users create postgres \
    --instance=hr-360-db \
    --password=[SECURE_PASSWORD]

Step 4: Get Connection String
  Command: gcloud sql instances describe hr-360-db \
    --format="value(connectionName)"

Step 5: Update Backend .env
  DATABASE_HOST=[CLOUD_SQL_IP]
  DATABASE_PORT=5432
  DATABASE_USER=postgres
  DATABASE_PASSWORD=[SECURE_PASSWORD]
  DATABASE_NAME=emergency_app

Step 6: Run Migrations
  Command: npm run migrate (from backend/)

Step 7: Deploy Updated Backend
  Command: gcloud builds submit backend/ \
    --tag gcr.io/hr-360-app/backend

Step 8: Verify Connection
  Command: curl https://backend-116253736511.us-central1.run.app/health

RISK MITIGATION
================================================================================

Risk: Database connection fails
Mitigation: 
  - Test connection before deploying
  - Keep fallback to JWT-based data
  - Have rollback procedure ready

Mitigation:

  - Monitor Redis health
  - Set up alerts

Mitigation:
  - Backup existing data
  - Test migration in staging
  - Have restore procedure

Risk: Downtime during deployment
Mitigation:
  - Use blue-green deployment
  - Deploy during low-traffic hours
  - Have quick rollback ready
